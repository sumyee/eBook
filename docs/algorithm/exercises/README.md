## LRU（最近最少使用）缓存机制

> 1. 需要提供`put`方法，用于写入不同的缓存数据，如果是同一站点重复写入，就覆盖;
> 2. 当缓存达到上限时， 调用`put`写入缓存之前, 要删除**最近最少使用的数据**；
> 3. 提供`get`方法，用于读取缓存数据，同时需要把被读取的数据，移动到**最近使用数据**；
> 4. 考虑到读取性能，希望`get`操作的复杂度是`O(1)`（简单理解就是，读取缓存时不能去遍历所有数据）
